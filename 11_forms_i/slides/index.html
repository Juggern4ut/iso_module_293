Modul 293

HTML Formulare I

Grundlagen für Benutzereingaben

Lernziele

  • Den Zweck des Elements form verstehen
  • action und method grob einordnen
  • Einfache Formulare mit Text-, Zahlenfeld und Submit-Button bauen

Wozu dienen Formulare?

  • Benutzereingaben auf einer Webseite erfassen
  • Zum Beispiel: Anmeldung, Suche, Kontaktformular
  • HTML sammelt die Daten, ein Server verarbeitet sie später

Das Element form

                    
                        <form action="/anfrage" method="post">
                          ...
                        </form>
  • form ist der Rahmen für Eingabefelder und Buttons
  • Alle Felder im Formular gehören logisch zusammen

action und method

  • action: Wohin werden die Daten gesendet?
  • method: Wie werden die Daten gesendet?
  • Für uns heute reicht: meist get oder post
                    
                        <form action="/registrieren" method="post">

Einfache Eingabefelder

                    
                        <input type="text" name="vorname" />
                        <input type="number" name="alter" />
  • type="text" für kurze Texte
  • type="number" für Zahlenwerte
  • name benennt das Feld für die Übertragung

Submit-Button

                    
                        <button type="submit">Absenden</button>
  • Der Button sendet das Formular ab
  • Ohne Submit-Button fehlt oft der offensichtliche Abschluss der Eingabe

Komplettes Mini-Beispiel

                    
                        <form action="/kursanmeldung" method="post">
                          <input type="text" name="vorname" />
                          <input type="text" name="nachname" />
                          <input type="number" name="alter" />
                          <button type="submit">Senden</button>
                        </form>

Typische Fehler

  • form ohne action oder method
  • Falschen type verwenden
  • Submit-Button vergessen
