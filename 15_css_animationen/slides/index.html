15 - CSS-Animationen

Bewegung direkt mit CSS

Lernziele

  • Verstehen, was CSS-Animationen grob sind
  • transition und animation unterscheiden
  • Ein sehr einfaches @keyframes-Beispiel lesen

Wozu Animationen?

  • Elemente können weicher erscheinen, verschwinden oder sich bewegen
  • Benutzer sehen schneller, was sich verändert
  • Animationen sollten sparsam und klar eingesetzt werden

Zwei Grundideen

  • transition: eine Änderung wird sanft animiert
  • animation: eine eigene Bewegungsfolge läuft automatisch ab

Einfach mit transition

                    
                        button {
                          background: steelblue;
                          transition: background 0.3s ease;
                        }

                        button:hover {
                          background: tomato;
                        }

Beim Hover wechselt die Farbe nicht hart, sondern weich.

Animation mit keyframes

                    
                        @keyframes schweben {
                          from { transform: translateY(0); }
                          to { transform: translateY(-10px); }
                        }

                        .box {
                          animation: schweben 1s infinite alternate;
                        }

Hier bewegt sich ein Element ständig leicht nach oben und unten.

Wichtige Begriffe

  • duration: Wie lange etwas dauert
  • delay: Wann etwas startet
  • ease: Wie schnell oder langsam es sich bewegt
  • infinite: Wiederholt sich ohne Ende

Merksätze

  • transition ist gut für Zustandswechsel
  • animation ist gut für eigene Abläufe mit @keyframes
  • Animationen sind ein Thema, das im nächsten Modul vertieft wird
