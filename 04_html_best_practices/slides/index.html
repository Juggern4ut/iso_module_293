04 - HTML Best Practices

Sauber, lesbar und zukunftssicher

Lernziele dieser Lektion

  • Verstehen, warum sauberes HTML wichtig ist.
  • HTML-Code lesbar und strukturiert schreiben können.
  • Typische Anfaengerfehler vermeiden.

Was sind Best Practices?

  • Bewährte Vorgehensweisen aus der Praxis.
  • Erleichtern Zusammenarbeit und Wartung.
  • Machen Webseiten robuster und verständlicher.

HTML ist Struktur

  • HTML beschreibt den Inhalt, nicht das Aussehen.
  • Layout und Design gehören in CSS.
  • Verwenden Sie HTML-Tags nach ihrer Bedeutung.

Semantische HTML-Tags

Semantische Tags beschreiben die Bedeutung des Inhalts.

Beispiele:

header, nav, main, section, footer

Keine Inline-Styles

  • HTML und CSS trennen.
  • Inline-Styles erschweren Wartung.
  • CSS gehört in eine externe Datei.

Kommentare sinnvoll nutzen

  • Erklären Struktur oder Zweck.
  • Nicht offensichtliches kommentieren.
