Responsive Design

Grundlagen für flexible Webseiten

13 - Responsiveness

HTML und CSS für verschiedene Bildschirmgrössen

Lernziele

  • Verstehen, was Responsive Design bedeutet
  • Media Queries mit max-width und min-width lesen und schreiben
  • Einfache responsive Layouts mit Prozentwerten umsetzen

Was bedeutet responsive?

  • Eine Webseite passt sich an verschiedene Bildschirmgrössen an
  • Inhalte bleiben lesbar und bedienbar auf Handy, Tablet und Desktop
  • Layout, Schriftgrössen und Abstände reagieren auf den verfügbaren Platz

Wichtige Grundlage im HTML

Damit der Browser auf Mobilgeräten korrekt skaliert:

                    
                        <meta
                          name="viewport"
                          content="width=device-width, initial-scale=1.0"
                        />

Ohne diese Zeile wirkt eine mobile Seite oft verkleinert oder unpraktisch.

Media Query mit max-width

Regeln gelten bis zu einer maximalen Breite.

                    
                        @media (max-width: 700px) {
                          body {
                            background-color: #1f1f1f;
                          }

                          nav a {
                            display: block;
                            margin-bottom: 12px;
                          }
                        }

max-width: 700px ist typisch für kleinere Bildschirme.

Media Query mit min-width

Regeln gelten ab einer minimalen Breite.

                    
                        @media (min-width: 900px) {
                          .content {
                            width: 80%;
                            margin: 0 auto;
                          }

                          .card {
                            display: inline-block;
                            width: 30%;
                          }
                        }

min-width eignet sich gut, um Layouts für grössere Geräte zu erweitern.

max-width oder min-width?

Ansatz Typische Idee
max-width Desktop zuerst, dann Anpassungen für kleinere Geräte
min-width Mobile zuerst, dann Erweiterungen für grössere Geräte

Für neue Projekte ist mobile first mit min-width oft einfacher.

Responsive Breiten mit Prozenten

                            
                                <main class="wrapper">
                                  <section class="left">Links</section>
                                  <section class="right">Rechts</section>
                                </main>
                            
                        
                            
                                .wrapper {
                                  width: 90%;
                                  margin: 0 auto;
                                }

                                .left,
                                .right {
                                  display: inline-block;
                                  width: 48%;
                                }

Prozentwerte machen Elemente flexibel, weil sie sich am verfügbaren Platz orientieren.

Einfacher Umbruch für kleine Screens

                    
                        @media (max-width: 700px) {
                          .left,
                          .right {
                            display: block;
                            width: 100%;
                          }
                        }

Zwei Spalten werden so auf kleinen Geräten untereinander dargestellt.

Best Practices

  • Mit einer einfachen Grundstruktur starten und erst dann erweitern
  • Breiten möglichst mit % oder max-width statt nur mit fixen Pixeln definieren
  • Inhalte auf echten kleinen und grossen Fenstern testen
  • Abstände, Schriftgrössen und Buttons auf Lesbarkeit prüfen

Merksätze

  • max-width: Regel gilt bis zu einer bestimmten Breite
  • min-width: Regel gilt ab einer bestimmten Breite
  • Prozentwerte helfen, damit Layouts nicht starr bleiben
